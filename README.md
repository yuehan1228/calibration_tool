# 雷达自动标定工具

基于 Flask 的雷达-GNSS 自动标定 Web 工具。通过上传 GNSS 和雷达坐标文件，输入装船机姿态参数，自动求解刚体变换矩阵并计算标定误差。

## 功能

- **坐标解析**：支持 CSV 和 TXT 格式的坐标文件
- **GNSS 格式支持**：
  - 局部坐标 (x, y, z)
  - 经纬度坐标 (B, L, H) - 自动转换为局部坐标
- **多坐标系变换**：支持旋转中心坐标系、臂架坐标系、溜筒坐标系
- **刚体变换求解**：基于 SVD 方法估计最优旋转平移矩阵
- **误差分析**：逐点欧氏距离误差，平均值和最大值
- **Web 界面**：浏览器直接操作，无需命令行

## 快速开始

### 环境要求

- Python 3.8+

### 安装

```bash
git clone https://github.com/your-username/calibration_tool.git
cd calibration_tool
pip install -r requirements.txt
```

### 运行

```bash
python app.py
```

浏览器访问 `http://localhost:5000`

### 使用步骤

1. **选择 GNSS 数据格式**：
   - 局部坐标：直接上传局部坐标系下的坐标文件
   - 经纬度：上传 BLH 格式文件，并输入 WGS84 到泊位坐标系的转换矩阵

2. **上传坐标文件**：
   - GNSS 坐标文件（CSV/TXT）
   - 雷达坐标文件（CSV/TXT）

3. **选择目标坐标系**：
   - 旋转中心坐标系（默认）
   - 臂架坐标系（需输入臂架参数）
   - 溜筒坐标系（需输入臂架参数和连接点参数）

4. **输入大机姿态参数**：
   - 回转角、俯仰角
   - 旋转中心坐标 (X, Y, Z)
   - 臂架长度、伸缩距离（臂架/溜筒坐标系）
   - 连接点距离（溜筒坐标系）

5. 点击"计算"，查看标定矩阵和误差

## 数据格式

### 局部坐标格式

列名必须为 `x`, `y`, `z`：

```csv
x,y,z
1234.567,5678.901,12.345
1235.123,5679.456,12.678
```

### 经纬度格式 (BLH)

支持多种列名识别：

| 数据 | 支持的列名 |
|------|-----------|
| 纬度 | B, b, lat, latitude, 纬度, Lat, Latitude |
| 经度 | L, l, lon, longitude, 经度, Lon, Longitude |
| 高度 | H, h, height, alt, altitude, 高度, Height |

示例：

```csv
B,L,H
30.123456,120.654321,15.5
30.123556,120.654421,15.6
```

### WGS84 转换矩阵格式

当选择经纬度格式时，需输入 4x4 齐次变换矩阵，仅支持 python 格式：

```
[[ 6.73269087e-01, -3.69559710e-02,  7.38473418e-01,  1.40248457e+04],
 [ 5.90597897e-01,  6.27782467e-01, -5.07033820e-01, -9.53467436e+03],
 [-4.44862738e-01,  7.77511044e-01,  4.44492648e-01, -6.37393720e+06],
 [ 0.00000000e+00,  0.00000000e+00,  0.00000000e+00,  1.00000000e+00]]
```

## 坐标系说明

### 旋转中心坐标系

以大机旋转中心为原点的坐标系，适用于安装在旋转中心附近的传感器。

### 臂架坐标系

以臂架末端为原点的坐标系，适用于安装在臂架上的传感器。

**参数**：
- 臂架长度：旋转中心到臂架末端的距离
- 伸缩距离：臂架的伸缩量

### 溜筒坐标系

以溜筒连接点为原点的坐标系，适用于毫米波雷达等安装在溜筒上的传感器。

**参数**：
- 臂架长度、伸缩距离
- 连接点距离：臂架末端到溜筒连接点的距离

## 技术细节

### BLH → 局部坐标转换流程

1. BLH（经纬度）→ WGS84 直角坐标 (x, y, z)
2. WGS84 直角坐标 → 泊位局部坐标（通过 4x4 变换矩阵）

### 刚体变换求解

使用 SVD（奇异值分解）方法求解最优旋转矩阵 R 和平移向量 t，使得：

```
min ||R * P_radar + t - P_gnss||
```
